=Kubernetes

# 개요
 [[container\|container]] 오케스트레이션 플랫폼.
 물리적으로 분단된 여러 서버(컴퓨팅 자원)들을 추상화하여 k8s라는 하나의 인터페이스로 마치 하나의 컴퓨터에서 돌아가듯 관리할 수 있게 해준다.

# 개발자 인터페이스
## control plane
여러 노드들을 감시, 관리.
etcd라는 저장소에 명세 ([[개발/yaml\|yaml]])를 기억해두고 24시간 루프를 돌며 실제 노드들이 명세와 같이 돌아가고 있는지 감시하며 관리한다. 워커 죽으면 다시 올리고 하는 식으로.
### kubectl
사용자가 k8s를 관리하기 위한 cli 도구. 이걸 이용해 어떤 pod를 몇개 올려두라고 명령하면 control plane이 그거에 맞게 관리한다.
## worker nodes
실제로 작업이 이루어지는 물리적 서버들.
### kubelet
각 서버에서 돌아가는 k8s 완장을 단 관리자. 위에서 특정 작업 명령이 들어오면 거기에 맞춰 컨테이너를 올리던가 한다.
## pod
worker node와는 별개로 추상화된 하나의 기능을 담당하는 단위. control에서 어느 node에 이 pod를 올릴지 결정하고, node에 pod를 올리는 것.
